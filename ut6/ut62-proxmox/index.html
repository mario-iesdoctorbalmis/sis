Proxmox

¿Qué es Proxmox?

  • Proxmox VE (Virtual Environment) es una plataforma de virtualización de código abierto que permite a los usuarios crear y gestionar máquinas virtuales y contenedores.
  • Se trata de virtualización de tipo 1 o bare-metal, lo que significa que se ejecuta directamente sobre el hardware del servidor, proporcionando un rendimiento eficiente y una gestión centralizada de los recursos.

Acceso a las MV o contenadores en Proxmox

Section titled “Acceso a las MV o contenadores en Proxmox”

  1. Acceso a la interfaz web de Proxmox:

    • Abre tu navegador web y dirígete a la URL: https://10.100.0.100:8006
    • Introduce las credenciales de usuario:
      • Usuario: usuX (X = número de usuario asignado)
      • Contraseña: xxxxxxxx

  2. Acceso a la máquina virtual:

    • Localiza la máquina virtual a la que deseas acceder.
    • Selecciona la máquina virtual y ve a la pestaña Consola.

  3. Uso de SPICE:

    • En la pestaña de Consola, selecciona SPICE como el tipo de consola.
    • Si no tienes el cliente SPICE instalado (Virt-viewer), Proxmox te proporcionará un enlace para descargarlo. Instálalo en tu máquina con Windows Server.
    • Una vez instalado, haz clic en el botón Conectar para abrir la consola SPICE y acceder a la máquina virtual.
      • El fichero de conexión que genera es de un único uso. Se debe generar otro en caso de cerrar sesión o ventana.

Prácticas de Windows Server en Proxmox

Section titled “Prácticas de Windows Server en Proxmox”

Para las prácticas necesitaremos un Servidor de Windows (WS) y un Windows 10 pro (W10). W10 será una máquina virtual en tu ordenador. Mientras que WS estará virtualizado en un servidor Proxmox

configuración para las prácticas

  • Usuario en WS: admin/Adm1nws
  • Usuario en W10/11: admin/Adm1n11

Práctica 1: Configuración de un servidor Windows Server en Proxmox

Section titled “Práctica 1: Configuración de un servidor Windows Server en Proxmox”

La máquina virtual de Windows Server ya está instalada con usuario admin/Admin12345. Sin embargo, es necesario realizar algunas configuraciones iniciales para asegurar su correcto funcionamiento y acceso.

Configuraciones iniciales

Section titled “Configuraciones iniciales”
  • Windows Server (WS)
  • Windows 10 (W10)
  • Cambiar el nombre del equipo a WS-apellidoNombre para una mejor identificación en la red.
    • Esto se debe hacer por temas de credenciales de objetos de Active Directory, ya que el nombre del equipo forma parte de la identidad del objeto en el dominio.
      • El protocolo de red de windows (netbios) utiliza el nombre del equipo para identificarlo en la red, y es fundamental que este nombre sea único y descriptivo para evitar conflictos y facilitar la administración.
  • Configurar una dirección IP estática para asegurar que el servidor siempre tenga la misma dirección en la red.
    • Usaremos el bloque de IPs privadas 10.100.YY.X/16
    • YY = número de IP
    • X = número de máquina virtual (MV) asignado
      • DAMA
        • 101 para el servidor WS
        • 102 para el cliente W10
        • 103 para el cliente W10-2 (en caso de necesitarlo)
        • Ejemplo: si tu IP es 10.100.43.1/16, la IP del servidor WS será 10.100.43.101/16
      • DAMB
        • 201 para el servidor WS
        • 202 para el cliente W10
        • 203 para el cliente W10-2 (en caso de necesitarlo)
        • Ejemplo: si tu IP es 10.100.22.1/16, la IP del servidor WS será 10.100.22.201/16
  • Cambiar el nombre del equipo a W10-YY-102 para una mejor identificación en la red.
    • Ejemplo: W10-43-102 para el cliente W10 de DAMA, W10-22-202 para el cliente W10 de DAMB.
  • Configurar la dirección IP estática siguiendo el mismo bloque de IPs privadas 10.100.YY.X/16 y asignando el número de máquina virtual correspondiente (En DAMA: 102 para el primer cliente, 103 para el segundo cliente en caso de necesitarlo).
  • Configurar la puerta de enlace y el servidor DNS apuntando a la IP del servidor WS para asegurar la conectividad y resolución de nombres en la red.
  • Cambiar el grupo de trabajo por el nombre del dominio.

Azure como alternativa a Proxmox

En caso de estar haciendo las FE y no tener acceso a Proxmox, se puede usar Azure para crear una máquina virtual de Windows Server. Sin embargo, es importante tener en cuenta que Azure es un servicio de pago. Debéis tener créditos con la cuenta gva para configurar el servidor y los clientes.

  • La configuración y gestión de la máquina virtual en Azure puede ser más compleja que en Proxmox, especialmente para aquellos que no están familiarizados con la plataforma de Microsoft.
  • PRECAUCIÓN:
    • Asegurate de apagar la máquina virtual en Azure cuando no la uses. Las máquinas virtuales en Azure pueden generar costos incluso cuando no están en uso, debido a los recursos que consumen (almacenamiento, IP pública, etc.).
    • Asegurate de apagarla/eliminarla después de terminar las prácticas para evitar cargos innecesarios.
